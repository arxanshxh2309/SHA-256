🔨 SHA-256 live

Type a message. Watch the padding bytes appear, the message schedule expand from 16 to 64 words, then step through all 64 rounds and see a..h evolve. Cells flash when they change.

All computation happens in your browser — open DevTools and you can watch console.assert verify against the FIPS 180-4 test vector.

1. Padded input

SHA-256 pads with 0x80, zeros, and the original message length in bits as a 64-bit big-endian number — so total length is always a multiple of 64 bytes.

2. Message schedule W[0..63]

First 16 words come from the message block. The other 48 are computed from earlier words via σ₀ and σ₁. The active word at the current round is highlighted.

3. Round formulas

T1 = h + Σ₁(e) + Ch(e,f,g) + K[t] + W[t]
T2 = Σ₀(a) + Maj(a,b,c)

h ← g, g ← f, f ← e
e ← d + T1
d ← c, c ← b, b ← a
a ← T1 + T2

// all arithmetic mod 2³², all rotations on 32-bit words
